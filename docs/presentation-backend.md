# Материалы к защите — Backend: как из точек получается смета

Документ для защиты: связный рассказ потока данных от ввода комнаты до сметы
с вилкой min/avg/max, без чтения кода. Все формулы — из `docs/estimation-rules.md`,
контракт ответа — из `docs/api.md`.

---

## Шаг 1 — Геометрия (`services/geometry_service.py`)

Вход: точки полигона, высота потолка, проёмы. Считаем в `Decimal` (prec=28),
чтобы не накапливать ошибку float на суммах.

| Величина | Формула | Функция |
|---|---|---|
| `floor_area` | площадь по формуле шнурования (Гаусса) | `floor_area()` |
| `ceiling_area` | зависит от формы потолка: плоский = `floor_area`, многоуровневый/мансардный скат — с надбавкой (#357) | `calculate_room_geometry()` |
| `perimeter` | Σ длин сторон | `perimeter()` / `side_lengths()` |
| `wall_area` | `perimeter · height − Σ(ширина·высота проёмов)` | `calculate_room_geometry()` |
| `door_width_sum` | Σ ширин дверей (для плинтуса; окна не вычитаем) | `calculate_room_geometry()` |

**Валидация проёмов** (`_validate_openings`, B1-8) — до расчёта, кидает `ValueError` →
эндпоинт отдаёт **422** с понятным сообщением:
- высота двери/окна ≤ высоты помещения;
- ширина проёма ≤ длины самой длинной стены;
- Σ площадей проёмов < площади стен (иначе `wall_area` ушёл бы в минус).

## Шаг 2 — Материалы и работы

### Материалы (`services/material_calc_service.py`)
`works` комнаты (отделочные группы `{enabled, finish}`) → список (материал, опорная площадь),
зависимые позиции добавляются автоматически (краска → грунтовка + шпаклёвка;
пол → плинтус; плитка → клей + затирка). Количество по `unit`:

- `л` (краски, грунт): `area · слои · норма_расхода · запас`;
- `м²`/`кг`: `area · норма · запас`;
- `м` (плинтус): `(perimeter − door_width_sum) · запас`;
- `рулон` (обои): `area · рулонов_на_м² · запас`.

Округление до упаковок здесь **не** делается — `pack_quantity` дробный; `ceil`
выполняется один раз на агрегации по квартире, иначе `1.3 + 1.3` дало бы 4 банки вместо 3.

### Работы (`services/labor_calc_service.py`)
`works` комнаты:
покраска/шпаклёвка стен и потолка — по площади; укладка пола/плитки — по площади;
электрика/сантехника — берут числа из `works.electric`/`works.plumbing`, дефолт от площади комнаты при пустом значении.
Каждой строке — `price_min/avg/max` за единицу и `total_*` = объём × цена.

## Шаг 3 — Цены (`services/price_aggregator_service.py`)

**Инвариант проекта:** цена берётся из парсера/кэша; seed — только fallback,
расчёт не падает при недоступности сети (`PARSER_LIVE_FETCH`, кэш наполняется
отдельно с РФ-IP). У каждой цены хранятся `source`, `source_url`, `region`,
`updated_at` — эти поля уходят в ответ построчно (B2-16), фронт рендерит ссылку на источник.

---

## Шаг 4 — Агрегация и вилка (`api/estimates.py` + `repair_coeffs_service.py`)

1. Группируем одинаковые материалы/работы по нескольким комнатам.
2. Материалы: `ceil` суммарного `pack_quantity` до упаковок → финальное количество.
3. Применяем **непредвиденные расходы**:

| Непредвиденные | min | avg | max |
|---|---|---|---|
| `CONTINGENCY` | 1.10 | 1.12 | 1.15 |

4. Вилка считается **отдельно** по материалам и работам (min/avg/max), затем суммируется
   в `total_*`. Построчные `total_avg` домножены на тот же `line_factor`
   (`непредв.avg`), чтобы сумма строк точно совпадала с `summary`.

Ответ — `EstimateResponse`: `summary` (9 чисел вилки), `geometry` (пол/потолок/стены/периметр),
`materials[]`, `labor[]` с источниками. Контракт — `docs/api.md`.

## Что готово по backend-расчётам (для слайда «сделано»)

- Геометрия произвольного полигона на `Decimal` + учёт и валидация проёмов.
- Материалы и работы из выбора отделки, нормы и запас из `estimation-rules.md`.
- Округление до фасовки на уровне квартиры (а не комнаты).
- непредвиденные расходы.
- Вилка min/avg/max отдельно по материалам и работам.
- Реальные цены как источник правды, seed-fallback, источник/ссылка/регион/дата у каждой цены.
- Региональность (город → цены), герметичные тесты (464 шт.), `ruff` зелёный.

## Тезисы для защиты

1. **Считаем в `Decimal`, не во float.** Деньги и площади — точные суммы без накопления
   ошибки округления; формула площади — шнурование, работает на любом невыпуклом полигоне.
2. **Смета — это вилка, а не одно число.** min/avg/max берутся из разброса реальных цен
   и считаются раздельно по материалам и работам — заказчик видит честный коридор стоимости.
3. **Реальные цены, но расчёт неубиваем.** Источник правды — парсер/кэш с пометкой источника
   и ссылкой; при недоступности сети тихий откат на seed, ответ всегда 200.
4. **Запас и фасовка — две ступени округления.** Сначала запас 8–12%, затем `ceil` до
   упаковок — и только после суммирования по всей квартире, иначе переплата на стыках.
5. **Валидация на входе.** Невозможные проёмы (дверь выше потолка, окно шире стены) —
   422 с понятным текстом, `wall_area` никогда не уходит в минус.
